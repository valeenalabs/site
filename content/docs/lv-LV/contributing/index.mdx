# Līdzdalība iekš HytaleModding

Mēs atzinīgi vērtējam kopienas ieguldījumus, lai palīdzētu uzlabot un paplašināt HytaleModding dokumentāciju. Neatkarīgi no tā, vai labojat drukas kļūdas, pievienojat jaunas apmācības vai sniedzat koda piemērus, jūsu līdzdalība ir vērtīga!

## Kā Piedalīties

### Zinošana par kļūdām

Ja atradi problēmu ar dokumentāciju, lūdzu, ziņo par to, atverot jaunu 'issue' mūsu [GitHub repozitorijā](https://github.com/HytaleModding/site/issues).

Ja varat pats apstrādāt problēmu, tad varat pieprasīt būt pievienotam un iesniegt PR.

### Pull Request Iesniegšana

1. Fork (nodali) repozitoriju GitHubā.
2. Klonē nodalīto repozitoriju uz savas ierīces.
3. Izveido jaunu branch (zaru) priekš izmaiņām

```bash
git checkout -b manu-izmainu-zars
```

4. Veic savas izmaiņas un commit (ievies) tās ar skaidru aprakstu. Mēs sekojam [konvenciālajai ieviešanai](https://www.conventionalcommits.org/en/v1.0.0/), Jūsu PR nebūs pieņemti, ja ieviešanas ziņa neseko šiem nosacījumiem.

```bash
git commit -m "Pievieno funkciju X dokumentācijai"
```

5. Push (Stum) izmaiņas savā nodalītajā repozitorijā.

```bash
git push origin manu-izmainu-zars
```

6. Atver Pull Request (Ievilkšanas Prasību) uz oriģinālo repozitoriju, aprakstot savas izmaiņas un kāpēc tās būtu jāiekļauj.
