# Contribuire a HytaleModding

Accogliamo con piacere i contributi della comunità per aiutare a  migliorare ed espandere la documentazione HytaleModding. Che tu stia correggendo errori di battitura, aggiungendo nuovi tutorial o fornendo esempi di codice, ogni tuo contributo è prezioso!

## Come Contribuire

### Segnalare Problemi

Se hai trovato un problema con la documentazione, per favore segnalalo aprendo una nuova issue sul nostro [repository GitHub](https://github.com/HytaleModding/site/issues).

Se potete lavorare in maniera indipendente a questo issue, potete chiedere che vi venga assegnato e inviare un PR.

### Invio di Pull Request

1. Forkare il repository su GitHub.
2. Clona il tuo fork nella tua macchina locale.
3. Crea un nuovo branch per le tue modifiche.

```bash
git checkout -b my-feature-branch
```

4. Effettua le tue modifiche e fai commit con messaggi chiari. Seguiamo [le conventional commits](https://www.conventionalcommits.org/en/v1.0.0/), la tua PR non sarà accettata se i tuoi messaggi di commit non seguono questa convenzione.

```bash
git commit -m "Aggiunta caratteristica X alla documentazione"
```

5. Fai push delle tue modifiche sul tuo fork.

```bash
git push origin my-feature-branch
```

6. Apri una PR (Pull Request) sul repository originale, descrivendo le tue modifiche e perché dovrebbero essere integrate.
