# Zu HytaleModding beitragen

Wir freuen uns über Beiträge der Community zur Verbesserung und Erweiterung der HytaleModding-Dokumentation. Egal ob du Rechtschreibfehler verbesserst, neue Tutorials hinzufügst oder Code-Beispiele bereitstellst, deine Beiträge sind wertvoll!

## Wie man beitragen kann

### Probleme melden

Wenn du ein Problem mit der Dokumentation gefunden hast, melde es bitte, indem du ein neues Problem (Issue) in unserem [GitHub Repository](https://github.com/HytaleModding/site/issues) öffnest.

Wenn du selbst an diesem Problem arbeiten kannst, kannst du darum bitten, dafür zugewiesen zu werden, und einen PR einreichen.

### Pull Request einreichen

1. Forke das Repository auf GitHub.
2. Klone deine lokale Fork auf deinen Rechner.
3. Erstelle eine neue Branch für deine Änderungen

```bash
git checkout -b my-feature-branch
```

4. Nimm deine Änderungen vor und committe sie mit einer klaren Nachricht. Wir folgen [conventional commits](https://www.conventionalcommits.org/en/v1.0.0/), deine PR wird nicht akzeptiert, wenn deine Commit-Nachrichten nicht dieser Konvention folgen.

```bash
git commit -m "Add feature X to documentation"
```

5. Push deine Änderungen in deine Fork.

```bash
git push origin my-feature-branch
```

6. Öffne eine Pull Request im ursprünglichen Repository und beschreibe deine Änderungen und warum sie zusammengeführt werden sollten.
