# Bidrag til HytaleModding

Vi værdsætter bidrag fra communityet, som hjælper med at forbedre og udvide dokumentationen for HytaleModding. Uanset om du retter stavefejl, tilføjer nye vejledninger eller deler kodeeksempler, er dine bidrag værdifulde!

## Hvordan du bidrager

### Rapportere fejl

Hvis du har fundet en fejl i dokumentationen, så vær venlig at rapportere det ved at oprette en ny issue på vores [GitHub repository](https://github.com/HytaleModding/site/issues).

Hvis du selv kan arbejde på fejlen, så kan du assigne dig selv til GitHub issuen og anmode om et pull request.

### Indsend pull requests

1. Fork repositoriet på GitHub.
2. Klon din fork til din lokale maskine.
3. Opret en ny branch med dine ændringer.

```bash
git checkout -b my-feature-branch
```

4. Lav dine ændringer og commit dem med klare beskeder om hvad der er ændret. Vi følger [konventionelle commits](https://www.conventionalcommits.org/en/v1.0.0/), din pull request vil ikke bliver accepteret, hvis dine commit beskeder ikke følger denne
   konvention.

```bash
git commit -m "Add feature x to documentation"
```

5. Push dine ændringer til din fork.

```bash
git push origin my-feature-branch
```

6. Åben en pull request på det oprindelige repository, hvor du beskriver dine ændringer og forklarer, hvorfor de bør merges.
